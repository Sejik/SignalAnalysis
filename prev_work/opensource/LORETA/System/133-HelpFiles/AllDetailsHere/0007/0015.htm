Abstract

Measures of linear dependence (coherence) and nonlinear dependence (phase synchronization) between any number of multivariate time series are defined. The measures are expressed as the sum of lagged dependence and instantaneous dependence. The measures are non-negative, and take the value zero only when there is independence of the pertinent type. These measures are defined in the frequency domain and are applicable to stationary and non-stationary time series. These new results extend and refine significantly those presented in a previous technical report (Pascual-Marqui 2007, arXiv:0706.1776 [stat.ME], http://arxiv.org/abs/0706.1776 ), and have been largely motivated by the seminal paper on linear feedback by Geweke (1982 JASA 77:304-313). One important field of application is neurophysiology, where the time series consist of electric neuronal activity at several brain locations. Coherence and phase synchronization are interpreted as “connectivity” between locations. However, any measure of dependence is highly contaminated with an instantaneous, non-physiological contribution due to volume conduction and low spatial resolution. The new techniques remove this confounding factor considerably. Moreover, the measures of dependence can be applied to any number of brain areas jointly, i.e. distributed cortical networks, whose activity can be estimated with eLORETA (Pascual-Marqui 2007, arXiv:0710.3341 [math-ph], http://arxiv.org/abs/0710.3341 ).

 

What this program does

 

You can compute measures of linear and nonlinear dependence between cortical regions of interest, i.e. between virtual electrodes placed on the cortex. A requirement is that the regions of interest be already defined, and that the corresponding ROI transformation matrix exists.

 

These measures are computed in the frequency domain, at a given time instant (in the sense of time-varying frequency analysis; see the time-varying cross-spectra module). For instance, you can compute the connectivity between cortical ROIs for gamma activity at 300 ms post stimulus. In order to use this module correctly, the user must specify three items:

1. The window function

2. The window width

3. The time frame of interest at which the spectral components will be analyzed.

 

Use the option "Smooth discrete freq range" to guarantee that the measures of connectivity be well defined. If the number of ERP files is small, or the frequency bands are too narrow, then the measures might be undefined, causing a program error. The problem of undefined measures is similar to trying to calculate a correlation coefficient with only one or two samples. Smoothing tries to avoid this problem.

 

If there are a total of “NDF” discrete frequencies or frequency bands, and if there are a total of “NROI” regions of interest, then the output files produced by this program have extensions:

 

1. Total linear connectivity: “*-ROIlinTotConn.txt”

2. Instantaneous linear connectivity: “*-ROIlinInstConn.txt”

3. Lagged linear connectivity: “*-ROIlinLagConn.txt”

 

The measures 1, 2, and 3 satisfy the relation: Total = Instantaneous + Lagged

 

The measures 1, 2, and 3 can be transformed to a “squared coherence” type value, taking values in the [0...1] range. See equations 19, 25, and 31 in the manuscript (http://arxiv.org/abs/0711.1455):

 

4. Total coherence: “*-ROItotCoh.txt”

5. Instantaneous coherence: “*-ROIInstCoh.txt”

6. Lagged coherence: “*-ROILagCoh.txt”

The measures 4, 5, and 6 DO NOT satisfy the relation: Total = Instantaneous + Lagged

 

7. Total nonlinear connectivity: “*-ROInonLinTotConn.txt”

8. Instantaneous nonlinear connectivity: “*-ROInonLinInstConn.txt”

9. Lagged nonlinear connectivity: “*-ROInonLinLagConn.txt”

 

The measures 7, 8, and 9 satisfy the relation: Total = Instantaneous + Lagged

 

The measures 7, 8, and 9 can be transformed to a “phase synchronization” type value, taking values in the [0...1] range. See equations 50, 51, and 52 in the manuscript (http://arxiv.org/abs/0711.1455):

 

10. Total coherence: “*-ROItotPhase.txt”

11. Instantaneous coherence: “*-ROIInstPhase.txt”

12. Lagged coherence: “*-ROILagPhase.txt”

The measures 10, 11, and 12 DO NOT satisfy the relation: Total = Instantaneous + Lagged

 

13. Total linear covariance: “*-ROIlinTotCov.txt”

14. Instantaneous linear covariance: “*-ROIlinInstCov.txt”

15. Lagged linear covariance: “*-ROIlinLagCov.txt”

 

The measures 13, 14, and 15, unlike the scaled coherences, are non-scaled log transformed squared covariances.

 

16. Total nonlinear covariance: “*-ROInonLinTotCov.txt”

17. Instantaneous nonlinear covariance: “*-ROInonLinInstCov.txt”

18. Lagged nonlinear covariance: “*-ROInonLinLagCov.txt”

 

The measures 16, 17, and 18, unlike the scaled phase synchronizations, are non-scaled log transformed squared covariances based on unit modulus Fourier transform coefficients (as explained in the manuscript (http://arxiv.org/abs/0711.1455)).

 

These output files are plain text files. Within, there are “NDF” matrices of dimension NROI* NROI, containing the connectivity values.

 

Statistical analysis for these connectivity measures are now included in the sLORETA/eLORETA SnPM module.

In addition, you can choose to produce (output) text files with the electric neuronal activity at the ROIs. This is equivalent to the activity recorded by “virtually” implanted electrodes in the cortex.

 

If you choose to produce (output) the “Squared activity values”, then the files will have number of columns equal to the number of ROIs, and number of rows equal to the number of time frames. The values in these files are non-negative, corresponding to “Squared activity values”. The file extension is “-SqrAct@Rois.txt”.

 

If you choose to produce (output) the “XYZ activity”, then the files will have number of columns equal to the number of ROIs, and number of rows equal to three times the number of time frames. The values in these files can be positive and negative, corresponding to the X, Y, and Z current density vector components at each ROI. The file extension in this case is “-XyzAct@Rois.txt”. Note that for XYZ activity:

- row #1 is time frame # 1 for the X-current density vector component.

- row #2 is time frame # 1 for the Y-current density vector component.

- row #3 is time frame # 1 for the Z-current density vector component.

- row #4 is time frame # 2 for the X-current density vector component.

- row #5 is time frame # 2 for the Y-current density vector component.

- row #6 is time frame # 2 for the Z-current density vector component.

- row #7 is time frame # 3 for the X-current density vector component.

- row #8 is time frame # 3 for the Y-current density vector component.

- row #9 is time frame # 3 for the Z-current density vector component.

- and so on...

 

In other words, for “XYZ activity” files, the columns correspond to the ROIs, and the rows correspond to triplicate time frames, such that the first three rows correspond to time frame # 1 (X, Y, and Z components respectively), the second three rows correspond to time frame # 2 (X, Y, and Z components respectively), and so on.

 

EEG outputs:

 

This version allows the computation of measures of similarity, NOT TO BE INTERPRETED AS CONNECTITY, between EEG signals. The files produced have similar extension to those above (measures 1 to 18), but with EEG substituting ROI. These files contain matrices of dimension equal to the number of electrodes, not to the number of regions of interest.
