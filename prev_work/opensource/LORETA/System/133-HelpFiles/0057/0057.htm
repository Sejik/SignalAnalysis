k-means clustering for LORETA files

 

The main input is a collection of LORETA files (*.slor). These are files of type:

array[1..NI,1..6239] of single (4ByteFloat)

where NI is number of LORETA images in the file (may correspond to different time frames, different frequencies, different components from ICAs, etc).

 

All files must have same number of images NI.

 

The main output is a LORETA with number of images equal to NC (number of clusters).

 

Note that NC need not be equal to the number of images in the input LORETA files
