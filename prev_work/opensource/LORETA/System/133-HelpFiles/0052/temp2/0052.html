Help on “Conjunction statistics for z-scores”


This utility combines two independent statistical analyses performed in the same cortical space as the sLORETA images. The two statistical analyses must be in the form of sLORETA files (*.slor) containing z-scores, i.e. standard Gaussian values.


As one example, consider two different subjects compared to the sLORETA-BRL norms. The two z-score files can be analyzed in terms of conjunction.


Conjunction analysis is explained in the papers:

1. Friston, K., Holmes, A., Price, C., Buchel, C., Worsley, K., 1999. Multisubject fMRI studies and conjunction analyses. NeuroImage 10, 385-396.

2. Nichols, T., Brett, M., Andersson, J., Wager, T., Poline, J.-B., 2004. Valid Conjunction Inference with the Minimum Statistic. NeuroImage 25, 653–660.

3. Friston, K.J., Penny,W.D., Glaser, D.E., 2005. Conjunction revisited. NeuroImage 25, 661-667.


Consider the previously mentioned simple example consisting of two z-score files (one for each subject). For each voxel and each frequency, there are two z-scores, denoted za and zb. Let the two tailed probabilities be denoted as pa and pb, respectively. These probabilities correspond to the Gaussian distribution. For instance, if za = 1.96, then pa = 0.05; and if zb = 2.57, then pb = 0.01. Then the probability for the conjunction pc is (see references above):

pc = [max(pa , pa)]2


In the example above, it follows that:

pc = [max(pa , pa)]2 = [max(0.05 , 0.01)]2 = [0.05]2 = 0.0025

which is very significant.


In fact, all that is needed for a significant conjunction at a given voxel and frequency is that the minimum z-score of the pair of values (or equivalently the maximum p) be zmin= 1.23, which gives:

pc = [0.224]2 = 0.05


This program computes for each voxel and frequency the conjunction probabilities, and outputs a file with:

-Log10pc

which is conveniently scaled to be interpreted as:

-Log10pc=1 corresponds to pc=0.1

-Log10pc=2 corresponds to pc=0.01

-Log10pc=3 corresponds to pc=0.001

-Log10pc=4 corresponds to pc=0.0001

And so on..


Also, this program computes for each voxel and frequency the conjunction z-score, which is conveniently interpreted as:

Conjunction z-score=1.95 corresponds to pc=0.05

Conjunction z-score=2.57 corresponds to pc=0.01

And so on..

As a second example, consider one subject that has been compared to the sLORETA-BRL norms, and that independently was also studied by SPECT imaging and compared to SPECT norms. If the z-score SPECT results can be converted to the sLORETA format, then the conjunction statistics can be calculated by this program. In this case, there are 6239 voxels and 87 discrete frequencies for the sLORETA-BRL z-scores, whereas for the SPECT z-score image there are only 6239 z-scores (there is no frequency information). Under these conditions, the program will perform a conjunction analysis using the same SPECT image for all frequencies. The resulting conjunction files will consist of conjunction p and z values for 6239 voxels and 87 discrete frequencies.

When the option “Sign sensitive output” is checked, the conjunction values (z and minus Log(p)) will be negative if the two z-scores analyzed for conjunction have different signs, and positive if the signs are the same. Otherwise (i.e. non-checked option “Sign sensitive output”), the conjunction values are always positive.


In this example, there are two files in “List A”, corresponding to two different subjects. The files in List A contain sLORETA-BRL z-scores. The two files in List B contain SPECT z-scores (at 6239 voxels), corresponding to the same subjects in List A. The program will perform a conjunction analysis for each subject, i.e. for each pair of files.


Note that in the example, the second pair consists of the files:

Second line, Subject 2, list A = 02-z-SubjWise.slor (containing 6239x87 BRL z-scores)

Second line, Subject 2, list B = D002_6239.slor (containing 6239 SPECT z-scores)


The single SPECT image is used for all sLORETA images (an image for each frequency, i.e. 87 frequencies) in the conjunction analysis. The resulting output files are:

02-z-SubjWise-Log10p.slor : corresponding to the minus base 10 Logarithm of the conjunction probabilities (values larger than 2 are highly significant).

02-z-SubjWise-zConj.slor : corresponding to the conjunction z-scores (values larger than 2.57 are highly significant)
